🚀 DEPLOYMENT STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Authenticate with Cloudflare
─────────────────────────────────────
npx wrangler login

This opens a browser for you to authorize Wrangler.

STEP 2: Test deployment
────────────────────────
npm run deploy:test

This deploys a tiny test page to verify everything works.
You'll get: https://jeremyrobards-site.pages.dev

STEP 3: Deploy your full site
──────────────────────────────
npm run deploy

This uploads your entire site to Cloudflare Pages.

STEP 4: Connect your custom domain
───────────────────────────────────
1. Go to: https://dash.cloudflare.com/
2. Click: Pages → jeremyrobards-site
3. Click: Custom domains
4. Click: "Set up a custom domain"
5. Enter: www.jeremyrobards.com
6. Click: "Activate domain"

Cloudflare will automatically:
- Create the DNS records (CNAME pointing to your Pages deployment)
- Set up SSL certificate
- Enable CDN

STEP 5: Add root domain redirect (optional)
────────────────────────────────────────────
If you want jeremyrobards.com (without www) to redirect:

1. In Cloudflare Pages custom domains, add: jeremyrobards.com
2. Cloudflare will create a redirect rule automatically

STEP 6: Disable GitHub Pages
─────────────────────────────
1. Go to: https://github.com/TheSeeker713/Jeremy-Robards-github.io
2. Click: Settings
3. Scroll to: Pages section
4. Click: "Unpublish site" or change Source to "None"

This stops GitHub from trying to build/deploy your site.
